# AI Security Assessment – Gandalf Challenge

## Overview

This project documents a practical assessment of Large Language Model (LLM) security risks using the Gandalf challenge platform.

The objective was to gain hands-on experience with prompt injection attacks, instruction manipulation, information disclosure risks, and AI guardrail effectiveness. The project focuses on understanding how adversarial inputs can influence AI systems and what security considerations organizations should address when deploying AI applications.

This assessment was completed as part of my AI Security, AI Governance, and AI Risk Management learning portfolio.

## Key Findings

### Prompt Injection

User-controlled instructions can significantly influence model responses and potentially alter intended system behavior.

### Context Manipulation

Conversation context may affect future responses and can be leveraged to influence AI outputs.

### Information Disclosure Risk

AI systems require layered controls to reduce the risk of exposing sensitive information.

### Guardrail Limitations

Prompt-based safeguards alone may not be sufficient for high-risk AI deployments.

### Importance of Human Oversight

Human review remains an important control for sensitive business processes involving AI.

## Risk Considerations

The following risks were observed during the assessment:

| Risk Area              | Description                                |
| ---------------------- | ------------------------------------------ |
| Prompt Injection       | Attempts to override intended instructions |
| Information Disclosure | Exposure of protected information          |
| Context Manipulation   | Influence through conversation history     |
| Output Integrity       | Changes to expected responses              |
| Guardrail Bypass       | Circumvention of intended restrictions     |

---

## Mitigation Recommendations

Organizations deploying AI systems should consider:

* Input validation controls
* Output monitoring
* Human oversight processes
* AI security testing
* Risk assessments before deployment
* Logging and monitoring of AI interactions
* Periodic AI red-team exercises
